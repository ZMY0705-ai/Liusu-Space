### 问题原因
前端代码中使用了错误的 localStorage key：
- ❌ **错误**：`localStorage.getItem('token')` 
- ✅ **正确**：`localStorage.getItem('access_token')`

#### 1. 前端修复 (`frontend/src/api/user.ts`)
```typescript

// 修复后

console.log('[上传头像] Token:', token ? `${token.substring(0, 20)}...` : 'null')

  return Promise.reject({ detail: '未登录，请先登录' })
```

## ❌ 如果仍然出现 401 错误

### 检查清单

#### 1. 确认已登录
- 查看浏览器控制台 → Application → Local Storage
- 确认存在 `access_token` key
- 值应该是类似 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` 的字符串

#### 2. 检查 Token 是否过期
- Token 默认有效期是 30 分钟
- 如果过期，请重新登录

#### 3. 查看后端日志
在后端终端中搜索：
- `[认证失败] 缺少 Authorization header` → 前端没有发送 Token
- `[认证失败] 无效的认证方案` → Token 格式错误
- `[认证失败] Token 解码返回 None` → Token 无效或已过期
- `[认证失败] JWT 错误` → Token 签名验证失败

#### 4. 手动测试 API
使用 Postman 或 curl 测试：

```bash
curl -X POST http://localhost:8000/users/avatar \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -F "file=@/path/to/avatar.jpg"
```

替换 `YOUR_TOKEN_HERE` 为实际的 token。

## 🎯 常见问题

### Q1: 为什么之前能获取用户信息，但上传头像失败？
**A**: 因为 `getCurrentUser()` 使用的是封装好的 `http.get()` 方法，它会自动从 store 中读取 token 并添加到请求头。而 `uploadAvatar()` 使用的是原生 `fetch`，需要手动处理 token。

### Q2: 如何查看当前存储的 token？
**A**: 在浏览器控制台中输入：
```javascript
console.log(localStorage.getItem('access_token'))
```

### Q3: Token 过期了怎么办？
**A**: 重新登录即可。系统会在登录成功后自动更新 token。

### Q4: 如何延长 Token 有效期？
**A**: 修改后端 `.env` 文件中的 `ACCESS_TOKEN_EXPIRE_MINUTES` 值（默认 30 分钟）。

## 📝 技术细节

### FormData 和 Content-Type
当使用 `FormData` 时，**不要手动设置** `Content-Type` header。浏览器会自动设置为：
```
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW
```

如果手动设置为 `multipart/form-data`，会缺少 `boundary` 参数，导致后端无法解析文件。

### FastAPI 的文件上传
FastAPI 使用 `UploadFile` 和 `File` 来处理文件上传：
```python
@router.post("/avatar")
def upload_avatar(
    file: UploadFile = File(...),  # 自动解析 multipart/form-data
    current_user: models.User = Depends(get_current_user)
):
    ...
```

### JWT 认证流程
1. 前端从 localStorage 读取 `access_token`
2. 添加到请求头：`Authorization: Bearer <token>`
3. 后端通过 `Header(None)` 获取 header
4. 解析 token，提取用户 ID
5. 从数据库查询用户信息
6. 返回用户对象给路由函数

# 作品发布流程修复 - 第三轮

## 修复时间
2026-05-07

## 核心需求

用户需要区分两条不同的作品发布链路：

### 链路1：储存作品（私密保存）
- **入口**：点击加号 → 选择"储存作品"
- **底部按钮**："保存草稿" + "保存作品"
- **保存草稿** (status=0)：保存在"我的草稿"，其他地方不可见
- **保存作品** (status=1, is_published=false)：保存在"我的作品"，但**不在首页显示**

### 链路2：发表作品（公开发布）
- **入口**：点击加号 → 选择"发表作品"
- **底部按钮**："储存草稿" + "发表作品"
- **储存草稿** (status=0)：保存在"我的草稿"，其他地方不可见
- **发表作品** (status=1, is_published=true)：保存在"我的作品"，**也在首页显示**

---

## 实现方案

### 数据库设计

使用两个字段来区分作品状态：

1. **`status`** (Integer):
   - `0`: 草稿
   - `1`: 已保存/已发表

2. **`published_at`** (DateTime):
   - `NULL`: 未真正发表（草稿或私密保存）
   - 有值: 已公开发表（首页可见）

### 可见性规则

| 位置 | 显示条件 |
|------|---------|
| **首页** | `published_at IS NOT NULL` |
| **我的作品** | `author_id = 当前用户 AND status = 1` |
| **我的草稿** | `author_id = 当前用户 AND status = 0` |
| **我的收藏** | 收藏表中的作品 |

## 修改内容

### 1. 后端修改

#### 1.1 Schema定义 (`backend/app/schemas.py`)

```python
class WorkCreate(BaseModel):
    title: str
    content: str
    cover_image: Optional[str] = None
    status: int = 0  # 0=draft, 1=published
    is_published: bool = False  # True=发表作品（首页可见），False=保存作品（仅我的作品可见）
```

#### 1.2 创建作品逻辑 (`backend/app/routers/works.py`)

```python
@router.post("/", response_model=schemas.WorkResponse)
def create_work(
    work: schemas.WorkCreate, 
    current_user: models.User = Depends(get_current_user), 
    db: Session = Depends(get_db)
):
    # 只有is_published=True时才设置published_at（首页可见）
    published_at = datetime.utcnow() if work.is_published else None
    
    db_work = models.Work(
        **work.model_dump(exclude={'is_published'}),  # 排除is_published字段
        author_id=current_user.id,
        published_at=published_at
    )
    db.add(db_work)
    db.commit()
    db.refresh(db_work)
    return db_work
```

#### 1.3 首页列表过滤 (`backend/app/routers/works.py`)

```python
@router.get("/", response_model=List[schemas.WorkResponse])
def list_published_works(skip: int = 0, limit: int = 100, db: Session = Depends(get_db)):
    from sqlalchemy.orm import joinedload
    # 只显示真正发表的作品（published_at不为空）
    works = db.query(models.Work).options(joinedload(models.Work.author)).filter(
        models.Work.published_at.isnot(None)
    ).offset(skip).limit(limit).all()
    return works
```

## 技术要点

### 1. 双字段状态管理
- `status`: 区分草稿(0)和非草稿(1)
- `published_at`: 区分是否真正发表(NULL vs 有值)

### 2. 路由参数传递
- `mode=draft`: 储存作品链路
- `mode=publish`: 发表作品链路
- `tab=works/drafts/favorites`: 切换"我的"页面标签

### 3. 条件渲染
使用 `v-if` / `v-else-if` 根据 `entryMode` 显示不同的按钮组合

### 4. 后端过滤
首页使用 `published_at IS NOT NULL` 而非 `status = 1` 来过滤

## 注意事项

1. **数据库兼容性**: 现有作品的 `published_at` 可能为NULL，需要手动更新已发表作品的 `published_at` 字段
2. **编辑模式**: 编辑已有作品时，`entryMode` 默认为 'publish'，保持原有行为
3. **向后兼容**: `is_published` 有默认值false，不影响其他调用createWork的地方
4. **用户体验**: 不同链路有不同的按钮文本，避免用户混淆

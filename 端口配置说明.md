## 🔧 配置文件位置

### 1. 前端端口配置
**文件**: `frontend/vite.config.ts`

```typescript
export default defineConfig({
  server: {
    port: 3002,              // 前端端口
    strictPort: true,        // 端口被占用时不自动切换
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:8000',  // 代理到后端
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

### 2. 后端端口配置
**启动命令**: 
```powershell
py -m uvicorn app.main:app --reload --port 8000
```

默认端口为 8000，如需修改可在启动时指定 `--port` 参数。

### 3. 数据库端口配置
**文件**: `backend/.env`

```env
DATABASE_URL=mysql+pymysql://root:123456@localhost:3306/liususpace?charset=utf8mb4
```

## 🔗 服务间通信

### 前端 → 后端 API 调用

前端通过 Vite 代理访问后端 API：

```
前端请求: http://localhost:3002/api/works/
     ↓ (Vite Proxy)
后端接收: http://127.0.0.1:8000/works/
```

**代理配置说明**：
- 前端代码中使用 `/api` 前缀
- Vite 自动将 `/api` 转发到后端
- 转发时自动移除 `/api` 前缀
- `changeOrigin: true` 解决跨域问题

### 示例代码

**前端调用** (`frontend/src/api/work.ts`)：
```typescript
export function getWorks(skip: number = 0, limit: number = 20) {
  return http.get<ApiResponse<Work[]>>('/works/', {  // 实际请求 /api/works/
    params: { skip, limit }
  })
}
```

**后端接收** (`backend/app/routers/works.py`)：
```python
@router.get("/works/")  # 接收 /works/ 请求
async def get_works(skip: int = 0, limit: int = 20):
    # 处理逻辑
```

## ⚠️ 端口占用处理

### 如果端口被占用

**前端端口 3002 被占用**：

1. **查找占用进程**：
```powershell
netstat -ano | findstr :3002
```

2. **终止进程**（假设 PID 为 12345）：
```powershell
taskkill /PID 12345 /F
```

3. **修改端口**（临时方案）：
```typescript
// vite.config.ts
server: {
  port: 3003,  // 改为其他端口
  strictPort: true
}
```

**后端端口 8000 被占用**：
```powershell
# 查找占用
netstat -ano | findstr :8000

# 终止进程
taskkill /PID <PID> /F
```

## 🌐 生产环境配置

### 前端部署

生产环境需要配置后端 API 地址：

```typescript
// frontend/src/api/http.ts
const http = axios.create({
  baseURL: process.env.NODE_ENV === 'production' 
    ? 'https://api.yourdomain.com'  // 生产环境 API 地址
    : '/api',                        // 开发环境使用代理
  timeout: 10000
})
```

### 后端部署

生产环境建议：
1. 使用 Nginx 反向代理
2. 配置 CORS 白名单
3. 启用 HTTPS

**Nginx 配置示例**：
```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # 前端静态文件
    location / {
        root /var/www/liususpace/frontend;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源（图片等）
    location /uploads/ {
        proxy_pass http://127.0.0.1:8000/uploads/;
    }
}
```

##  常见问题

### Q1: 前端无法连接后端？
**A**: 检查以下配置：
- Vite proxy 配置是否正确
- 后端是否在 8000 端口运行
- 浏览器控制台是否有 CORS 错误

### Q2: 图片无法加载？
**A**: 检查：
- 后端 `/uploads` 静态文件服务是否正常
- 图片路径是否正确（应包含 `/uploads/covers/`）
- 浏览器 Network 面板查看 404 错误

### Q3: 端口总是被占用？
**A**: 解决方案：
- 关闭之前的服务进程
- 使用不同的端口
- 检查是否有其他应用占用

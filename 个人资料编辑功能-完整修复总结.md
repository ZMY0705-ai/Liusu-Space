#### B. 新增工具函数
```typescript
// 获取完整头像URL（处理相对路径）
const getFullAvatarUrl = (url: string | undefined) => {
  if (!url) return ''
  // 如果是相对路径，拼接后端地址
  if (url.startsWith('/')) {
    return `${import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000'}${url}`
  }
  return url
}
```

#### C. 添加 onActivated 钩子（关键修复）
```typescript
import { ref, onMounted, onActivated } from 'vue'

// 组件名称（用于 keep-alive）
defineOptions({
  name: 'MeCenter'
})

// ... 其他代码 ...

// 当页面被激活时（从其他页面返回），重新获取用户信息
onActivated(async () => {
  console.log('[MeCenter] 页面被激活，刷新用户信息')
  await authStore.fetchUser()
})
```

**作用**:
- 当用户从"编辑资料"页面返回"我的"页面时
- `onActivated` 会自动触发
- 调用 `authStore.fetchUser()` 从后端重新获取最新的用户信息
- 确保页面上显示的是最新数据

#### E. 启用 keep-alive (`App.vue`)
```vue
<template>
  <div class="app-container">
    <MobileLayout v-if="layout === 'mobile'">
      <router-view v-slot="{ Component }">
        <keep-alive :include="['MeCenter']">
          <component :is="Component" />
        </keep-alive>
      </router-view>
    </MobileLayout>
    <div v-else>
      <router-view v-slot="{ Component }">
        <keep-alive :include="['MeCenter']">
          <component :is="Component" />
        </keep-alive>
      </router-view>
    </div>
  </div>
</template>
```

**为什么需要 keep-alive?**
- Vue Router 默认会在路由切换时销毁组件
- 使用 `keep-alive` 可以缓存 MeCenter 组件
- 这样从编辑页面返回时，`onActivated` 钩子才会触发
- 如果不使用 keep-alive，每次都会执行 `onMounted`，而不是 `onActivated`

## 🎯 技术要点总结

### 1. JWT 认证
- **Token 存储**: `localStorage.setItem('access_token', token)`
- **Token 读取**: `localStorage.getItem('access_token')`
- **请求头格式**: `Authorization: Bearer <token>`

### 2. FormData 上传
- **不要手动设置** `Content-Type`
- 浏览器会自动添加正确的 `boundary`
- 使用原生 `fetch` API 而非 Axios（更简单）

### 3. Vue 3 生命周期
- **`onMounted`**: 组件首次挂载时执行
- **`onActivated`**: keep-alive 缓存的组件被激活时执行
- **配合使用**: `keep-alive` + `onActivated` = 完美的页面刷新机制

### 4. Pinia Store 响应式
- `authStore.user` 是响应式的
- 调用 `authStore.fetchUser()` 会更新 store
- 所有使用该数据的组件会自动重新渲染

### 5. 条件渲染最佳实践
```vue
<!-- ✅ 正确：使用 v-if / v-else-if -->
<p v-if="user?.bio">{{ user.bio }}</p>
<p v-else-if="user?.signature">{{ user.signature }}</p>

<!-- ❌ 错误：两个都会显示 -->
<p v-if="user?.bio">{{ user.bio }}</p>
<p v-if="user?.signature">{{ user.signature }}</p>
```

### 6. 隐私控制实现
```vue
<!-- 只有 is_real_name_public 为 true 且 real_name 有值时才显示 -->
<p v-if="user?.is_real_name_public && user?.real_name">
  📝 {{ user.real_name }}
</p>
```

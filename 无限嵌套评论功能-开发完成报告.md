## 🔧 技术实现细节

### 1. 数据库设计

#### WorkComment表结构
```python
class WorkComment(Base):
    __tablename__ = "work_comments"
    
    id = Column(BIGINT(unsigned=True), primary_key=True)
    work_id = Column(BIGINT(unsigned=True), ForeignKey("works.id"))
    user_id = Column(BIGINT(unsigned=True), ForeignKey("users.id"))
    content = Column(String(500))
    parent_id = Column(BIGINT(unsigned=True), ForeignKey("work_comments.id"))  # 新增
    created_at = Column(DateTime, server_default=func.now())
    
    # 新增关系
    parent = relationship("WorkComment", remote_side=[id])
    replies = relationship("WorkComment", backref="parent_comment", remote_side=[parent_id])
```

#### 迁移SQL
```sql
ALTER TABLE work_comments

ADD CONSTRAINT fk_work_comment_parent FOREIGN KEY (parent_id) REFERENCES work_comments(id);

CREATE INDEX idx_work_comments_parent_id ON work_comments(parent_id);
```

### 2. 后端API实现

#### 发表评论接口
```python
@router.post("/works/{work_id}/comments")
def add_work_comment(
    work_id: int, 
    comment: schemas.WorkCommentCreate,  # 包含parent_id
    current_user: models.User = Depends(get_current_user), 
    db: Session = Depends(get_db)
):
    # 验证父评论是否存在
    if comment.parent_id:
        parent_comment = db.query(models.WorkComment).filter(
            models.WorkComment.id == comment.parent_id,
            models.WorkComment.work_id == work_id
        ).first()
        if not parent_comment:
            raise HTTPException(status_code=404, detail="Parent comment not found")
    
    # 创建评论
    db_comment = models.WorkComment(
        content=comment.content,
        work_id=work_id,
        user_id=current_user.id,
        parent_id=comment.parent_id  # 保存parent_id
    )
    db.add(db_comment)
    db.commit()
    
    return {"message": "Comment added", "comment_id": db_comment.id}
```

#### 获取评论列表（树形结构）
```python
@router.get("/works/{work_id}/comments")
def list_work_comments(work_id: int, db: Session = Depends(get_db)):
    # 获取所有评论
    comments = db.query(models.WorkComment).options(
        joinedload(models.WorkComment.user)
    ).filter(
        models.WorkComment.work_id == work_id
    ).all()
    
    # 递归构建树形结构
    def build_comment_tree(comments_list, parent_id=None):
        tree = []
        for comment in comments_list:
            if comment.parent_id == parent_id:
                comment_dict = {
                    "id": comment.id,
                    "content": comment.content,
                    "user": {...},
                    "replies": build_comment_tree(comments_list, comment.id)  # 递归
                }
                tree.append(comment_dict)
        return tree
    
    return build_comment_tree(comments, parent_id=None)
```

## ✨ 核心特性

### 1. 无限嵌套
- 通过 `parent_id` 字段建立评论间的父子关系
- 递归组件实现UI上的无限层级显示
- 每层缩进12px，左侧有灰色引导线

### 2. 树形结构
- 后端一次性返回完整树形结构
- 减少前端计算和网络请求次数
- 适合评论数量适中的场景（<1000条）

### 3. 登录鉴权
- 发表评论前检查JWT token
- 未登录时跳转到登录页
- 保留当前URL，登录成功后自动返回

### 4. 实时UI更新
- 提交评论后通过事件通知父组件
- 重新加载评论列表，不刷新整个页面
- 用户体验流畅

### 5. 代码复用
- CommentSection组件支持作品和论坛两种类型
- 通过Props区分 `targetType`
- 减少重复代码

## ⚠️ 注意事项

### 1. 性能考虑
**当前实现**:
- 一次性加载所有评论
- 适合评论数量 < 1000的场景

**优化建议**（如果评论很多）:
- 分页加载顶级评论（每页20条）
- 懒加载子评论（点击"展开x条回复"时加载）
- 虚拟滚动长列表

### 2. 层级限制
**当前实现**:
- 理论上支持无限层级
- 实际受屏幕宽度限制

**建议**:
- UI上限制最大显示层级（如5层）
- 超过层级的回复可以合并显示为"还有x条回复"

### 3. 删除评论

- 未实现删除功能

**如需实现**:

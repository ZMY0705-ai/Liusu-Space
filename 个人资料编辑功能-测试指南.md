## 🐛 常见问题排查

### 问题 1：头像上传返回 404

**原因：** 后端静态文件服务未正确配置

**解决方案：**
检查 `backend/app/main.py` 中是否有以下代码：
```python
UPLOAD_DIR = "uploads"
os.makedirs(UPLOAD_DIR, exist_ok=True)
app.mount("/uploads", StaticFiles(directory=UPLOAD_DIR), name="uploads")
```

### 问题 2：保存资料时返回 500 错误

**排查步骤：**
1. 查看后端终端的错误日志
2. 检查数据库中 `users` 表是否包含新字段
3. 确认 SQLAlchemy 模型已更新（`backend/app/models.py`）

**常见错误：**
```
sqlalchemy.exc.OperationalError: (MySQLdb._exceptions.OperationalError) 
(1054, "Unknown column 'bio' in 'field list'")
```
→ 说明数据库迁移未执行，请重新执行 SQL 脚本

### 问题 3：前端 TypeScript 报错

**错误示例：**
```
Property 'bio' does not exist on type 'UserUpdate'
```

**解决方案：**
确认 `frontend/src/types/api.ts` 中已添加新字段：
```typescript
export interface UserUpdate {
  nickname?: string
  bio?: string
  is_real_name_public?: boolean
  is_major_public?: boolean
  // ... 其他字段
}
```

然后重启前端开发服务器：
```bash
# Ctrl+C 停止
npm run dev
```

### 问题 4：头像无法显示

**检查清单：**
1. 头像文件是否存在于 `backend/uploads/avatars/` 目录
2. 后端是否正常运行在 8000 端口
3. 前端环境变量 `VITE_API_BASE_URL` 是否正确配置

**调试方法：**
在浏览器中直接访问头像 URL：
```
http://localhost:8000/uploads/avatars/<文件名>.jpg
```
如果能看到图片，说明后端服务正常；如果 404，检查文件路径。

## 📊 数据库字段说明

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `bio` | TEXT | NULL | 个性签名，最多 150 字 |
| `is_real_name_public` | SMALLINT | 0 | 是否公开真实姓名（0=否，1=是） |
| `is_major_public` | SMALLINT | 0 | 是否公开专业/学院（0=否，1=是） |

**注意：**
- `bio` 字段在后端有长度校验（≤150 字符）
- 隐私开关字段使用 `SmallInteger`，0 表示关闭，1 表示开启
- 现有用户的 `bio` 会自动从 `signature` 字段复制（如果存在）

## ⚠️ 本次修复的三个核心问题

### 问题1：保存后的作品消失在"我的作品"中
**原因**：后端更新作品时没有正确处理 `is_published` 字段，导致 `published_at` 未被设置

### 问题2：发表作品中只有"储存草稿"按钮，"发表作品"按钮消失
**原因**：条件判断使用了 `v-if` / `v-else-if`，当 `entryMode` 不是 'draft' 也不是 'publish' 时（编辑模式），两个按钮都不显示

### 问题3：草稿编辑应该有三个按钮
**原因**：没有区分编辑模式和新建模式的按钮逻辑

// 提交表单（关键修改：接收isPublished参数）
async function handleSubmit(isPublished: boolean = false) {
  // ... 验证逻辑 ...
  
  const workData = {
    title: formData.value.title,
    content: formData.value.content,
    cover_image: coverUrl,
    status: formData.value.status,
    is_published: isPublished  // 由调用者决定
  }
  
  if (isEdit.value && workId.value) {
    await updateWork(workId.value, workData)
  } else {
    await createWork(workData)
  }
  
  // 跳转逻辑
  if (isPublished && formData.value.status === 1) {
    router.push({ path: '/home', query: { refresh: Date.now().toString() } })
  } else if (formData.value.status === 0) {
    router.push({ path: '/me', query: { tab: 'drafts' } })
  } else {
    router.push({ path: '/me', query: { tab: 'works' } })
  }
}

// 三个处理函数分别传递不同的isPublished值
async function handleSaveDraft() {
  formData.value.status = 0
  await handleSubmit(false)  // 草稿
}

async function handleSaveWork() {
  formData.value.status = 1
  await handleSubmit(false)  // 保存但不发表
}

async function handlePublish() {
  formData.value.status = 1
  await handleSubmit(true)   // 保存到首页
}
```

@router.put("/{work_id}", response_model=schemas.WorkResponse)
def update_work(work_id: int, work_update: schemas.WorkUpdate, ...):
    work = db.query(models.Work).filter(...).first()
    
    # 处理is_published字段
    if work_update.is_published is not None:
        if work_update.is_published and work.published_at is None:
            work.published_at = datetime.utcnow()
        elif not work_update.is_published:
            work.published_at = None
    
    # 更新其他字段（排除is_published）
    for field, value in work_update.model_dump(
        exclude_unset=True, 
        exclude={'is_published'}
    ).items():
        setattr(work, field, value)
    
    db.commit()
    db.refresh(work)
    return work
```

## 🔑 关键技术要点

### 1. 双字段状态管理
- **status**: 区分草稿(0)和非草稿(1)
- **published_at**: 区分是否真正发表(NULL vs 有值)
- **is_published**: 前端传递的操作意图（临时参数，不存入数据库）

### 2. 三种模式区分
- **isEdit=true**: 编辑已有作品，显示三个按钮
- **entryMode='draft'**: 新建-储存作品，显示两个按钮（保存草稿+保存作品）
- **entryMode='publish'**: 新建-发表作品，显示两个按钮（储存草稿+发表作品）

### 3. 按钮颜色语义
- **灰色**: 草稿相关操作
- **浅绿色 (#BCDDBE)**: 保存作品（私密）
- **深绿色 (#64A386)**: 发表作品（公开）

### 4. 后端过滤逻辑
- **首页**: `published_at IS NOT NULL`
- **我的作品**: `author_id = 当前用户 AND status = 1`
- **我的草稿**: `author_id = 当前用户 AND status = 0`

## 1. 项目基本信息

### 1.1 项目概述

**项目名称**: 流俗地 (Liusu Space)  
**项目类型**: 校园文学创作与交流平台  
**产品定位**: "个人创作工具 + 内容社区 + 校园论坛" 的融合体

### 1.2 核心功能

| 功能模块 | 描述 | 优先级 | 状态 |
|---------|------|--------|------|
| 用户系统 | 注册、登录、JWT认证、个人信息管理 | P0 | ✅ 已完成 |
| 作品管理 | 创作、发布、编辑、删除文学作品 | P0 | ✅ 已完成 |
| 互动功能 | 点赞、收藏、评论作品 | P1 | ✅ 已完成 |
| 论坛社区 | 发帖、回复、楼中楼评论 | P1 | ✅ 已完成 |
| 消息通知 | 点赞、评论、收藏、回复通知 | P2 | ✅ 已完成 |

### 1.3 目标用户

- 校园文学爱好者
- 需要私密创作空间的学生
- 希望分享作品的创作者
- 喜欢文学讨论的读者

### 1.4 视觉风格

- **主色调**: #64A386（森林绿，传递静谧感）
- **辅助色**: #CDE2EB（天空蓝）、#BCDDBE（浅绿）
- **背景色**: #FAF9F6（纸张米色，适合长文阅读）
- **风格**: 文艺清新、简约优雅

## 7. API接口文档

### 7.1 基础信息

- **后端地址**: http://127.0.0.1:8000
- **API文档**: http://127.0.0.1:8000/docs
- **认证方式**: Bearer Token (JWT)
- **数据格式**: JSON

### 7.2 用户模块 (/users)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| POST | /users/register | 用户注册 | 否 | UserRegister |
| POST | /users/login | 用户登录 | 否 | UserLogin |
| GET | /users/me | 获取当前用户信息 | 是 | - |
| PUT | /users/me | 更新当前用户信息 | 是 | UserUpdate |

### 7.3 作品模块 (/works)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| POST | /works/ | 创建作品 | 是 | WorkCreate |
| GET | /works/ | 获取已发布作品列表 | 否 | 查询参数: skip, limit |
| GET | /works/{work_id} | 获取作品详情 | 否 | - |
| PUT | /works/{work_id} | 更新作品 | 是 | WorkUpdate |
| DELETE | /works/{work_id} | 删除作品 | 是 | - |
| POST | /works/upload-cover | 上传封面图片 | 否 | FormData (file) |

**查询参数**:
- `skip`: 偏移量，默认0
- `limit`: 每页数量，默认20

### 7.4 互动模块 (/interactions)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| POST | /works/{work_id}/comments | 评论作品 | 是 | WorkCommentCreate |
| GET | /works/{work_id}/comments | 获取作品评论列表 | 否 | - |
| POST | /works/{work_id}/like | 点赞/取消点赞 | 是 | - |
| POST | /works/{work_id}/favorite | 收藏/取消收藏 | 是 | - |
| GET | /works/{work_id}/interaction-status | 获取用户互动状态 | 是 | - |

### 7.5 论坛模块 (/forum)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| POST | /forum/posts | 创建帖子 | 是 | PostCreate |
| GET | /forum/posts | 获取帖子列表 | 否 | 查询参数: skip, limit |
| GET | /forum/posts/{post_id} | 获取帖子详情 | 否 | - |
| POST | /forum/posts/{post_id}/comments | 回复帖子 | 是 | PostCommentCreate |
| GET | /forum/posts/{post_id}/comments | 获取帖子回复列表 | 否 | - |

### 7.6 通知模块 (/notifications)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| GET | /notifications/ | 获取我的通知列表 | 是 | - |
| PUT | /notifications/{notification_id}/read | 标记通知为已读 | 是 | - |
| PUT | /notifications/read-all | 全部标记为已读 | 是 | - |

## 8. 数据库设计

### 8.1 数据库信息

- **数据库名**: liususpace
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **引擎**: InnoDB
- **连接URL**: mysql+pymysql://root:123456@localhost:3306/liususpace

### 8.2 表结构（8张表）

#### 8.2.1 users（用户表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 用户ID | PRIMARY KEY, AUTO_INCREMENT |
| account | VARCHAR(50) | 账号 | UNIQUE, NOT NULL |
| password_hash | VARCHAR(255) | 密码哈希 | NOT NULL |
| nickname | VARCHAR(50) | 昵称 | NOT NULL |
| real_name | VARCHAR(50) | 真实姓名 | NULL |
| student_id | VARCHAR(20) | 学号 | NULL |
| major | VARCHAR(100) | 专业 | NULL |
| avatar | VARCHAR(255) | 头像URL | NULL |
| signature | TEXT | 个性签名 | NULL |
| role | TINYINT | 角色(0:普通用户, 1:管理员) | DEFAULT 0 |
| created_at | DATETIME | 创建时间 | NOT NULL |
| updated_at | DATETIME | 更新时间 | NOT NULL |

#### 8.2.2 works（作品表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 作品ID | PRIMARY KEY, AUTO_INCREMENT |
| author_id | BIGINT UNSIGNED | 作者ID | FOREIGN KEY -> users.id |
| title | VARCHAR(100) | 标题 | NOT NULL |
| content | TEXT | 内容 | NOT NULL |
| cover_image | VARCHAR(255) | 封面图片URL | NULL |
| status | TINYINT | 状态(0:草稿, 1:已发布) | DEFAULT 0 |
| view_count | BIGINT UNSIGNED | 浏览量 | DEFAULT 0 |
| like_count | INTEGER | 点赞数 | DEFAULT 0 |
| comment_count | INTEGER | 评论数 | DEFAULT 0 |
| favorite_count | INTEGER | 收藏数 | DEFAULT 0 |
| created_at | DATETIME | 创建时间 | NOT NULL |
| updated_at | DATETIME | 更新时间 | NOT NULL |
| published_at | DATETIME | 发布时间 | NULL |

#### 8.2.3 work_comments（作品评论表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 评论ID | PRIMARY KEY, AUTO_INCREMENT |
| work_id | BIGINT UNSIGNED | 作品ID | FOREIGN KEY -> works.id |
| user_id | BIGINT UNSIGNED | 用户ID | FOREIGN KEY -> users.id |
| content | TEXT | 评论内容 | NOT NULL |
| created_at | DATETIME | 创建时间 | NOT NULL |

## 11. 开发规范

### 11.1 代码规范

#### TypeScript
- 使用严格的类型检查
- 所有API响应必须有类型定义
- 避免使用 any 类型

#### Vue 3
- 使用 Composition API (`<script setup>`)
- 组件命名使用 PascalCase
- 文件命名与组件名一致

#### 命名规范
- 变量/函数: camelCase
- 组件: PascalCase
- 常量: UPPER_SNAKE_CASE
- 文件: kebab-case 或 PascalCase

### 11.2 Git规范

#### 分支策略
- `main`: 主分支（生产环境）
- `develop`: 开发分支
- `feature/*`: 功能分支
- `bugfix/*`: 修复分支

#### Commit规范
```
<type>(<scope>): <subject>

type: feat | fix | docs | style | refactor | test | chore
scope: 模块名（可选）
subject: 简短描述
```

**示例**:
```
feat(editor): 添加封面图片上传功能
fix(api): 修复作品列表分页错误
docs(readme): 更新项目文档
```

### 11.3 API规范

#### 响应格式
```typescript
interface ApiResponse<T> {
  code: number        // 状态码: 200成功, 4xx客户端错误, 5xx服务器错误
  message: string     // 消息
  data: T            // 数据
}
```

#### 错误处理
- 使用HTTP状态码表示错误类型
- 返回清晰的错误消息
- 前端统一处理401未授权

### 11.4 样式规范

#### CSS变量
使用CSS变量定义主题色：
```css
:root {
  --color-primary: #64A386;
  --color-secondary: #CDE2EB;
  --color-bg: #FAF9F6;
  --color-text: #333;
}
```

#### 响应式断点
- 移动端: < 768px
- 平板: 768px - 1024px
- 桌面: > 1024px

## 12. 常见问题

### 12.1 开发问题

#### Q1: 前端无法连接后端？
**A**: 
- 检查后端是否在8000端口运行
- 检查Vite代理配置是否正确
- 查看浏览器控制台是否有CORS错误

#### Q2: 图片无法加载？
**A**:
- 检查后端uploads目录是否存在
- 确认图片URL路径正确（应包含/uploads/covers/）
- 检查浏览器Network面板404错误

#### Q3: 登录失败？
**A**:
- 确认数据库中有测试账号
- 检查bcrypt版本是否为4.0.1
- 查看后端日志错误信息

#### Q4: 端口被占用？
**A**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :3002

# 终止进程
taskkill /PID <PID> /F
```

### 12.2 部署问题

#### Q5: 生产环境如何配置？
**A**:
- 修改 `.env` 中的 `SECRET_KEY`
- 配置CORS白名单
- 使用Nginx反向代理
- 启用HTTPS

#### Q6: 数据库迁移？
**A**:
- SQLAlchemy自动创建表（首次启动）
- 使用Alembic进行数据库迁移（推荐）
- 备份数据库：`mysqldump -u root -p liususpace > backup.sql`

### 12.3 性能问题

#### Q7: 首页加载慢？
**A**:
- 实现图片懒加载
- 添加分页加载
- 使用CDN加速静态资源
- 启用Gzip压缩

#### Q8: 大文本编辑卡顿？
**A**:
- 实现自动保存到本地
- 使用虚拟滚动
- 添加防抖/节流

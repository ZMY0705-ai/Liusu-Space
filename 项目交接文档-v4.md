## 1. 项目概述

### 1.1 项目基本信息

**项目名称**: 流俗地 (Liusu Space)  
**项目类型**: 校园文学创作与交流平台  
**产品定位**: "个人创作工具 + 内容社区 + 校园论坛" 的融合体  
**目标用户**: 校园文学爱好者、创作者、读者

### 1.2 核心功能模块

| 功能模块 | 描述 | 优先级 | 状态 |
|---------|------|--------|------|
| 👤 用户系统 | 注册、登录、JWT认证、个人信息管理 | P0 | ✅ 已完成 |
| 📚 作品管理 | 创作、发布、编辑、删除文学作品 | P0 | ✅ 已完成 |
|  互动功能 | 点赞、收藏、评论作品 | P1 | ✅ 已完成 |
|  论坛社区 | 发帖、回复、楼中楼评论 | P1 | ✅ 已完成 |
| 🔔 消息通知 | 点赞、评论、收藏、回复通知 | P2 | ✅ 已完成 |
| ✏️ 个人资料 | 头像上传、个性签名、隐私设置 | P1 | ✅ 已完成 |

### 1.3 视觉风格

- **主色调**: #64A386（抹茶绿/森林绿，传递静谧感）
- **辅助色**: #F5B342（柠檬黄）、#A8D5A1（浅绿）
- **背景色**: #FAF9F6（纸张米色，适合长文阅读）
- **设计风格**: 治愈系扁平插画风、大圆角、柔和阴影、留白充足
- **UI组件库**: Vant 4（移动端优先）

## 5. API接口文档

### 5.1 基础信息

- **后端地址**: http://127.0.0.1:8000
- **API文档**: http://127.0.0.1:8000/docs (Swagger UI)
- **前端地址**: http://localhost:3002
- **认证方式**: Bearer Token (JWT)
- **数据格式**: JSON
- **CORS**: 允许所有来源（生产环境需限制）

### 5.2 用户模块 (/users)

| 方法 | 路径 | 说明 | 认证 | 请求体 |
|------|------|------|------|--------|
| POST | `/users/register` | 用户注册 | ❌ | UserRegister |
| POST | `/users/login` | 用户登录 | ❌ | UserLogin |
| GET | `/users/me` | 获取当前用户信息 | ✅ | - |
| PUT | `/users/me` | 更新当前用户信息 | ✅ | UserUpdate |
| POST | `/users/avatar` | 上传头像 | ✅ | FormData (file) |

**请求体示例 - 注册**:
```json
{
  "account": "testuser",
  "password": "test123",
  "nickname": "测试用户",
  "real_name": "张三",
  "student_id": "20240001",
  "major": "计算机科学与技术"
}
```

**新增字段（个人资料）**:
- `bio` - 个性签名（TEXT，最多150字）
- `is_real_name_public` - 是否公开姓名（SmallInteger，默认0）
- `is_major_public` - 是否公开专业（SmallInteger，默认0）

**查询参数**:

### 5.7 认证机制


**响应**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

**使用Token**:
```bash
GET /users/me
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 6. 数据库设计

### 6.1 数据库信息

- **数据库名**: liususpace
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **引擎**: InnoDB
- **连接URL**: `mysql+pymysql://root:123456@localhost:3306/liususpace?charset=utf8mb4`

### 6.2 表结构（8张表）

#### 6.2.1 users（用户表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 用户ID | PRIMARY KEY, AUTO_INCREMENT |
| account | VARCHAR(50) | 账号 | UNIQUE, NOT NULL |
| password_hash | VARCHAR(255) | 密码哈希 | NOT NULL |
| nickname | VARCHAR(30) | 昵称 | NOT NULL |
| avatar | VARCHAR(500) | 头像URL | NULL |
| signature | VARCHAR(100) | 个性签名（旧） | NULL |
| **bio** | **TEXT** | **个性签名（新）** | **NULL** |
| real_name | VARCHAR(20) | 真实姓名 | NULL |
| student_id | VARCHAR(20) | 学号 | NULL |
| major | VARCHAR(50) | 专业 | NULL |
| **is_real_name_public** | **SMALLINT** | **是否公开姓名** | **DEFAULT 0** |
| **is_major_public** | **SMALLINT** | **是否公开专业** | **DEFAULT 0** |
| role | SMALLINT | 角色(0:普通用户, 1:管理员) | DEFAULT 0 |
| created_at | DATETIME | 创建时间 | server_default=now() |
| updated_at | DATETIME | 更新时间 | server_default=now(), onupdate=now() |

**新增字段说明**:
- `bio`: TEXT类型，替代原有的signature字段，支持更长文本
- `is_real_name_public`: 0=不公开，1=公开（默认0）
- `is_major_public`: 0=不公开，1=公开（默认0）

#### 6.2.2 works（作品表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 作品ID | PRIMARY KEY, AUTO_INCREMENT |
| author_id | BIGINT UNSIGNED | 作者ID | FOREIGN KEY -> users.id |
| title | VARCHAR(100) | 标题 | NOT NULL |
| content | TEXT | 内容 | NOT NULL |
| cover_image | VARCHAR(255) | 封面图片URL | NULL |
| status | TINYINT | 状态(0:草稿, 1:已发布) | DEFAULT 0 |
| view_count | BIGINT UNSIGNED | 浏览量 | DEFAULT 0 |
| like_count | INTEGER | 点赞数 | DEFAULT 0 |
| comment_count | INTEGER | 评论数 | DEFAULT 0 |
| favorite_count | INTEGER | 收藏数 | DEFAULT 0 |
| created_at | DATETIME | 创建时间 | NOT NULL |
| updated_at | DATETIME | 更新时间 | NOT NULL |
| published_at | DATETIME | 发布时间 | NULL |

#### 6.2.3 work_comments（作品评论表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 评论ID | PRIMARY KEY, AUTO_INCREMENT |
| work_id | BIGINT UNSIGNED | 作品ID | FOREIGN KEY -> works.id |
| user_id | BIGINT UNSIGNED | 用户ID | FOREIGN KEY -> users.id |
| content | TEXT | 评论内容 | NOT NULL |
| created_at | DATETIME | 创建时间 | NOT NULL |

#### 6.2.4 work_likes（作品点赞表）

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| work_id | BIGINT UNSIGNED | 作品ID | PRIMARY KEY, FOREIGN KEY |
| user_id | BIGINT UNSIGNED | 用户ID | PRIMARY KEY, FOREIGN KEY |

## 9. 开发规范

### 9.1 代码规范

#### TypeScript
- ✅ 使用严格的类型检查（`strict: true`）
- ✅ 所有API响应必须有类型定义
- ✅ 避免使用 `any` 类型
- ✅ 使用 interface 定义数据结构

#### Vue 3
- ✅ 使用 Composition API (`<script setup>`)
- ✅ 组件命名使用 PascalCase
- ✅ 文件命名与组件名一致
- ✅ 使用 `defineOptions` 定义组件名称（用于 keep-alive）

#### 命名规范
- 变量/函数: `camelCase`（如 `getUserInfo`）
- 组件: `PascalCase`（如 `WorkCard.vue`）
- 常量: `UPPER_SNAKE_CASE`（如 `MAX_FILE_SIZE`）
- 文件: `kebab-case` 或 `PascalCase`

### 9.2 Git规范

#### 分支策略
- `main`: 主分支（生产环境）
- `develop`: 开发分支
- `feature/*`: 功能分支（如 `feature/profile-edit`）
- `bugfix/*`: 修复分支（如 `bugfix/avatar-upload`）

#### Commit规范
```
<type>(<scope>): <subject>

type: feat | fix | docs | style | refactor | test | chore
scope: 模块名（可选）
subject: 简短描述（不超过50字符）
```

**示例**:
```
feat(profile): 添加个人资料编辑功能
fix(upload): 修复头像上传401错误
docs(readme): 更新项目交接文档
refactor(mecenter): 重构个人中心为小红书风格
```

```typescript
interface ApiResponse<T> {
```

## 10. 常见问题与故障排查

### 10.1 开发问题

#### Q1: 前端无法连接后端？
**症状**: 浏览器控制台显示 `ERR_CONNECTION_REFUSED` 或 CORS 错误

**解决方法**:
1. 检查后端是否在8000端口运行
2. 检查Vite代理配置（`vite.config.ts`）
3. 查看后端终端是否有报错
4. 确认CORS配置允许前端域名

```typescript
// vite.config.ts 中的代理配置
server: {
  proxy: {
    '/api': {
      target: 'http://127.0.0.1:8000',
      changeOrigin: true
    }
  }
}
```

#### Q2: 头像上传失败（401 Unauthorized）？
**症状**: 上传头像时返回 401 错误

**原因**: localStorage key 错误

**解决方法**:
- ✅ 已修复：使用 `localStorage.getItem('access_token')` 而非 `token`
- 检查文件：`frontend/src/api/user.ts`

#### Q3: 个人中心数据不刷新？
**症状**: 编辑资料后返回个人中心，新数据不显示

**原因**: 缺少 keep-alive 和 onActivated 钩子

**解决方法**:
- ✅ 已修复：
  1. `App.vue` 添加 `<keep-alive :include="['MeCenter']">`
  2. `MeCenter.vue` 添加 `onActivated` 钩子调用 `authStore.fetchUser()`
  3. `EditProfile.vue` 保存成功后调用 `authStore.fetchUser()`

#### Q4: 图片无法加载？
**症状**: 头像或封面显示空白

**解决方法**:
1. 检查后端 `uploads` 目录是否存在且有读写权限
2. 确认图片URL路径正确（应包含 `/uploads/avatars/` 或 `/uploads/covers/`）
3. 检查浏览器Network面板是否有404错误
4. 确认后端静态文件服务配置正确

```python
# backend/app/main.py
app.mount("/uploads", StaticFiles(directory="uploads"), name="uploads")
```

#### Q5: 登录失败？
**症状**: 输入正确账号密码仍提示错误

**解决方法**:
1. 确认数据库中有测试账号
2. 检查 bcrypt 版本是否为 4.0.1（不能是 5.x）
3. 查看后端日志错误信息
4. 确认 `.env` 中数据库连接正确

```bash
# 检查 bcrypt 版本
py -m pip show bcrypt

# 降级到正确版本
py -m pip install bcrypt==4.0.1
```

#### Q6: 端口被占用？
**症状**: 启动时报错 `Port 3002 is already in use`

**解决方法**:
```powershell
# 查找占用端口的进程
netstat -ano | findstr :3002

# 终止进程（替换<PID>为实际进程ID）
taskkill /PID <PID> /F

# 或者修改端口（vite.config.ts）
server: {
  port: 3003  # 改为其他端口
}
```
